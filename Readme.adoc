== Cloning the repository

LitterBox is included as a submodule.
You either need to clone the repository using
[source,bash]
----
git clone --recurse-submodules URL
----
or run
[source,bash]
----
git submodule update --init
----
afterwards when having cloned without `--recurse-submodules`.


== Building

[source,bash]
----
mvn install -DskipTests  # <1>
cd litterbox-web
mvn package spring-boot:repackage
----
<1> Needed for Maven to be able to find the LitterBox dependency in litterbox-web.

Afterwards, you can start the litterbox-web server using
[source,bash]
----
java -jar litterbox-web/target/litterbox-web-${version}.jar
----

Starting the application directly via your IDE from the main class in litterbox-web should also work.

.LitterBox submodule changes
[NOTE]
--
LitterBox is included as a Git submodule.
If the build fails because it cannot find LitterBox or some of its classes/methods, you might have to run
[souce,bash]
----
git submodule update --init
mvn install -DskipTests
----
again to update the submodule.
--


=== Building a container image

[source,bash]
----
mvn spring-boot:build-image
# or for an image that precompiles using GraalVM
mvn spring-boot:build-image -Pnative
----


=== Building a GraalVM binary

[source,bash]
----
mvn native:compile -Pnative
----


== Linting

=== Licence headers

You can automatically add missing licence headers using
[source,bash]
----
mvn license:format
----


=== Code formatting

Automatically format the code and sort imports using
[source,bash]
----
mvn spotless:apply
----


=== Checkstyle

Run the Checkstyle linter using
[source,bash]
----
mvn checkstyle:check
----
